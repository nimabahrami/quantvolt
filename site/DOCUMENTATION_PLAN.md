# QuantVolt complete documentation plan

Status: planned — implementation has not started for this expansion.

## Objective

Document the entire supported QuantVolt surface with no abbreviated or implied setup. Every
public workflow must show how all inputs are created, how the function is called, what the
typed result contains, representative output, validation behavior, and where to continue.

Current generated inventory: 294 public symbols across 13 subpackages. The expanded inventory
must additionally cover the top-level facade, exception hierarchy, `quantvolt.testing`,
serialization helpers, and optional/provider-specific behavior.

## Definition of complete

Every public class, enum, constant, protocol and function must have:

- [ ] Correct fully qualified name and import path.
- [ ] Source-derived signature with defaults and type annotations.
- [ ] Plain-language purpose and when to use it.
- [ ] Complete parameter table including units, accepted ranges and conventions.
- [ ] Return/result table describing every field and its units.
- [ ] Complete executable example with every prerequisite object constructed in the snippet.
- [ ] Deterministic representative output generated by that example.
- [ ] At least one invalid-input example for important validation boundaries.
- [ ] Raised exception types and the conditions that trigger each one.
- [ ] Notes about mutation, determinism, numerical method and performance where relevant.
- [ ] Links to related domain types, lower-level kernels and next workflow step.
- [ ] Source link and tests that substantiate the documented behavior.
- [ ] Search keywords and placement in the left-hand navigation hierarchy.

An example is not complete if it references unexplained names such as `market`, `curve`,
`positions`, `columns`, `scenarios`, `plant` or `prices`. These must be constructed in the same
example or linked to a directly preceding, copy-pasteable setup block.

## Recommended implementation order

1. Coverage infrastructure and shared executable fixtures.
2. Models, curves and numerics—the prerequisites used everywhere else.
3. Pricing, portfolio and risk—the primary financial workflow.
4. PPAs, hedging, assets and curve models—the physical/structured workflow.
5. Statistics, market, workflow and data adapters.
6. Exceptions, testing and cross-module tutorials.
7. Full automated release gate and final editorial pass.
